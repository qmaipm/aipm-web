## 1. 三个底层设计原则(贯穿全站)

1. **用户视角先于产品视角** — 每个页面首屏先讲"用户为什么问题失眠",再讲"我们的产品是什么"。技术词(Agent、Skill、Tool Use)不在首屏出现。
2. **政治海拔守恒** — 对标 Palantir 政府白皮书。克制、留白、不替对方下结论;政府向页面的主 CTA 永远是"指导/共建"姿态,不是"购买/试用"姿态。
3. **每页只承担一个论点** — 对标 Anthropic 主页结构。一屏一个核心信息,论证短促,转化路径清晰。

- **必须使用**:共建 / 同担 / 承接 / 沉淀 / 实证 / 路径 / 范式 / 命题 / 共识 / 重构 / 破解 / 释放 / 贵区

### 13.2 技术 SEO 基线(开发验收项)

- HTML5 语义化(`<article>` / `<nav>` / `<aside>` 各得其所)
- **Schema.org JSON-LD**:Organization(全站)/ Product(产品页)/ Article(行业研究)/ BreadcrumbList(内页)/ FAQPage(预约体验页)/ Person(团队页)
- 自动生成 `sitemap.xml` + `robots.txt`
- 域名 301 规范化(www / 非 www 选一为主)
- 全站 HTTPS + HTTP/2 或 HTTP/3
- **Core Web Vitals 目标**:LCP < 2.5s · CLS < 0.1 · INP < 200ms
- 移动端 Lighthouse 分数 ≥ 90
- URL 使用英文/拼音 slug(避免中文)
- 图片懒加载 + WebP/AVIF + alt 文本

### 13.3 收录与外链

- 提交:百度站长 / 搜狗站长 / 360 站长 / Bing Webmaster / Google Search Console
- 公众号双发(每篇行业研究在公众号同步发布,带回链)
- 争取被中物协、克而瑞物管、亿翰物业评论等行业媒体引用

### 13.4 每页必备的 SEO 元数据(checklist)

- 唯一的 H1(包含核心关键词)
- title(50-60 字符,含品牌)
- meta description(150-160 字符,带 CTA 暗示)
- og:image · og:title · og:description(社交分享)
- canonical 链接
