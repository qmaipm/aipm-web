# 页面改版设计参考（以 /products/fmclaw 为样板）

> 这份文档沉淀了本轮 **FMClaw 产品页重设计**里确立的设计与工程约定。
> 后续给**其它页面**做改版时，照这份做，保证与全站、与已改好的页面保持一致。
> 配合根目录 `DESIGN.md`（全站设计系统）一起看：`DESIGN.md` 定基线，本文件定**改版手法**。

样板文件：`app/(site)/products/fmclaw/page.tsx` + `app/(site)/products/fmclaw/page.css`

---

## 0. 三条硬约束（任何页面都不能破）

1. **单一字体**：全站只用一种字族（Noto Sans SC / 系统栈），**禁止任何 `font-family` 声明**（page.css 里看到残留就删）。性格全靠**字号 / 字重 / 字距**拉开，不靠换字体。无版权风险。
2. **蓝绿 VI**：强调色只用蓝绿 `#0070FF → #12B98A`（及其安全变体）。**不得出现暖色**（金 `#BE7A2E`、紫、粉）。暖琥珀 `--human` 只保留给"人做最后决定"这一种真实语义。
3. **标题不带句号**；标题尽量**一行**（见 §3）。

---

## 1. 配色：冷调"平台"色板（产品/平台页适用）

全站默认是**暖米纸底**；但**产品页 / 平台页**可以走一套更冷、更"操作台"的色板，与暖色内容页区分开，同时仍在蓝绿 VI 之内。把这套 token 收在页面根 wrapper 上（见 §6），不污染全站。

```css
.fmx{
  --fm-ink:#14201C;  --fm-ink-2:#46524D; --fm-mut:#7C857F; --fm-faint:#A7AFAB;
  --fm-line:#E6EAE8; --fm-mist:#ECF3F1;  --fm-paper:#FFFFFF;
  --fm-core:#0B1714;                      /* 单一暗场（signature dark） */
  --fm-grad:linear-gradient(115deg,#0070FF,#12B98A);
}
```

- 浅底两档：`--fm-paper`(白) / `--fm-mist`(冷雾灰) 交替做斑马纹（§4）。
- 暗场只用一处：`--fm-core`（near-black green），留给全页最有分量的一段。
- 蓝绿渐变文字统一用 `.grad`（带 `-webkit-background-clip:text` + 回退色）。

> 暖色内容页（about/news/solutions 等）**不必**改成冷色——保持暖纸底即可，只要遵守 §0/§3/§4。冷色板是给"平台/产品/技术"气质的页面的选项。

---

## 2. 字体与排版

- **Display（大标题）**：`font-weight:800`，`letter-spacing:-.03em`，大字号（hero 最大 ~72px，h2 最大 ~40px）。
- **Body**：常规字重，行高 1.6–1.7。
- **Label / eyebrow**：小字 + 大写拉丁 + 宽字距（`letter-spacing:.14em–.22em` + `text-transform:uppercase`），做"系统/平台"的标签感。
- **大数字 / 序号**：用蓝绿渐变（`.grad` 同款 clip），`font-variant-numeric:tabular-nums`（累加/对齐不抖）。

---

## 3. 标题与换行规则（本轮反复出现的需求）

**总则：能一行的就一行，不要为排版硬换行。**

| 元素 | 规则 |
|---|---|
| Hero 大标题（h1） | **唯一例外**，允许两行；可用 `<br/>` 在语义点断行（如把关键词放到第二行 + 蓝绿渐变强调） |
| 章节标题（h2） | **一行**。手法：`max-width:none; white-space:nowrap`，并加窄屏兜底 `@media(max-width:880px){white-space:normal}` 防溢出 |
| 副标题 / 小字 / 说明 | **去掉 `max-width:Nch` 限制**（设 `none`），必要时字号收一档，让它在桌面端尽量一行 |
| 所有标题 | **去掉结尾句号 `。`** |

**导致"该一行却换行"的三大元凶，排查顺序：**
1. 元素自身 `max-width:Nch` 太窄 → 放开。
2. **父容器** `max-width`（如 `.fm-cfg{max-width:900px}`）在卡宽度 → 放开父容器。
3. 旁边的**标签列**占太宽（如 `grid-template-columns:230px 1fr`）→ 收窄标签列、给正文让宽。

> 长到满宽也放不下的（两句话的副标题），就是"尽量"的极限，到两行可接受；要严格一行只能压文案（需先得到许可）。

**强制换行 `<br/>` 的处理**：正文里的 `<br/>` 一律删（让它自然排版）；示意框/小卡里的多行标签可保留（删了会溢出）。

## 4. 段落背景：斑马纹（明暗交替）

**问题**：相邻两段同底色 → 分不清是两段。
**做法**：段与段**背景交替**，相邻两段必不同色；全页只留**一处暗场**作签名打断。

样板的节奏（每段都和上下不同）：

```
Hero(白) → 段A(雾灰) → 段B(白) → 段C(雾灰) → 段D(白)
        → 暗场 core(深墨绿，唯一暗场) → 使命时刻(雾灰) → CTA(白)
```

- 雾灰 `--fm-mist` 调到**一眼能看出**与白不同（样板用 `#ECF3F1`）。
- 用 `.fm-band` / `.fm-band.mist` 两个类在 markup 里手动交替（不要用 `nth-of-type`，因为中间有暗场/特殊段打断，nth 会错）。

**加底色后必查**：原本"白卡片放在白底上"的组件，挪到雾灰段后要确认对比；反过来"雾灰元素放到雾灰段"会糊掉（见 §5 them/us 的坑）。

---

## 5. 可复用的组件模式

### 5.1 对比卡 them vs us（"通用平台 vs 我们"）
**坑**：早期把左卡底色设成雾灰，整段又是雾灰 → 左卡和页面糊在一起、看着像空了一块。
**正确做法**：做成**两张独立卡**，不靠"和段底不同"取得对比，而是各自有边框圆角，**在任何底色上都立得住**：
- them = 白卡（`--fm-paper` + 细边），us = 蓝绿淡底卡（`linear-gradient(rgba(18,185,138,.08), paper)` + 蓝绿边）。
- 中间一个 `vs` 圆徽章（带轻投影）。

### 5.2 编号梯 capability ladder（能力 01/02/03）
内容**本身有序**（第一/二/三能力）时才用编号。编号当作视觉锚点 + 进度感，不是装饰。

### 5.3 公式 / 等式（把"核心洞察"做成版式）
`Skill × 行业 Data × 行业 Tools = 一条工作流`——把抽象主张拆成可读的 token 行（结果项用蓝绿实底高亮）。比一段话更有记忆点。

### 5.4 把"散文墙"重构成"列表 + 判词"（解决"字太抖"）
**症状**：暗底上几段灰色正文堆在一起，没落点、字细又暗 → 读着"抖"。
**重构**：抽出每段的主题句当**小标题**，原文留作支撑句，做成：
- N 条"要点行"：**大号蓝绿渐变序号** + 白色小标题 + 提亮的支撑文字（`#AEBAB5`），细分割线隔开；
- 末尾一句**判词 callout**（白色大字 + 蓝绿竖条 `::before`），关键词用 `.grad` 高亮。

### 5.5 浅色"使命时刻"（参考首页深色版，转浅色）
首页的使命是**深色签名时刻**（点阵 canvas）。在子页复刻时**转成浅色**：
- 居中大字（参考首页字号份量）+ 上方英文 eyebrow + 下方支撑句；
- 背后**浅色科技氛围**：一层柔和**蓝绿径向微光**（`filter:blur(82px); opacity:.55`）+ 淡蓝图网格，居中向四周淡出；
- 放在雾灰段，夹在暗场和白色 CTA 之间，明暗节奏顺。

### 5.6 蓝图网格 + 微光（轻量科技氛围，替代重 canvas）
不想上 canvas 时，用纯 CSS 出"科技底"：
```css
.fm-grid{position:absolute;inset:0;pointer-events:none;
  background-image:linear-gradient(to right,rgba(12,139,130,.16) 1px,transparent 1px),
                   linear-gradient(to bottom,rgba(12,139,130,.16) 1px,transparent 1px);
  background-size:54px 54px;
  mask-image:radial-gradient(125% 90% at 80% 0%,#000,transparent 62%)}
```

## 6. CSS 工程约定

- **页面级样式全部收进页面 wrapper 类**（如 `<main className="fmx">` + 所有规则前缀 `.fm-*` / `.fmx`）。**禁止给 `:root` / `body` / 通用类加全局覆盖**（早期 fmclaw 在 `:root` 改 `--vi-grad` 泄漏到全站——已废弃，别再犯）。
- 页面 token 定义在 wrapper 上（`.fmx{--fm-*:...}`），不挂 `:root`。
- **复用全站类**（`.btn`/`.btn-primary`/`.endcta` 等）保持按钮/CTA 全站一致；只为本页新组件起 `.fm-*` 类。
- **渐变文字**统一带回退：`@supports not (background-clip:text){color:var(--teal)}`。
- **边框渐变**：只在"单边"用 `border-image`（其余边宽度 0 才不会糊四边）；要"顶部渐变条"用 `::before` 绝对定位一条，别用 `border-image` 配 `overflow:hidden`（会裁掉伸出的箭头等）。
- **质量底线**（每页都要有）：`prefers-reduced-motion` 降级、`:focus-visible` 可见焦点、移动端 ≤600px 无横向溢出、触摸目标 ≥44px。

## 7. 改版检查清单（每页改完逐条过）

- [ ] 无任何 `font-family`；无暖色（`#BE7A2E`/金/紫/粉）；强调只蓝绿。
- [ ] 所有标题无结尾句号；h2 一行（窄屏兜底换行）；副标题尽量一行。
- [ ] 段落背景**斑马纹**，相邻两段不同色；全页 ≤1 处暗场。
- [ ] 加底色后，卡片/元素与新段底**对比 OK**（没有"同色糊一起"）。
- [ ] 页面样式**全部 scoped**，无 `:root`/`body`/通用类全局泄漏。
- [ ] 渐变文字有回退；动效有 `prefers-reduced-motion` 降级。
- [ ] `npx tsc --noEmit` 通过；该页 `curl` 返回 200；窄屏无横向溢出。
